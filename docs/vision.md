# 设计愿景

nuonuo-pet 的核心不是“聊天机器人”，而是一个有身份、有情绪、有成长轨迹的电子宠物。

## 核心抽象

- **Device**：具体硬件设备
- **Pet**：宠物实体
- **Species Template**：物种模板，例如猫、猴子、鼠、恐龙
- **Theme Pack**：外观主题，例如白天/夜间/节日
- **Memory**：短期记忆、长期记忆、事件记忆
- **Emotion**：情绪值与状态标签
- **Growth**：成长阶段、经验、习惯与偏好
- **Model Route**：模型提供方与路由规则

## 第一版要解决的问题

1. 设备如何注册和绑定
2. 宠物如何在离线/弱网时维持基本可用
3. 资源如何以槽位方式导入并校验
4. 模型如何按成本、速度、效果路由
5. 记忆如何分层而不是无限堆叠

## 约束

- ESP32 端资源有限
- 不假设用户一定有固定云后端
- 所有扩展都要留出降级路径
