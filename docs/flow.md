# 交互流程

## 最小闭环

1. 设备上电，进入连接态
2. 设备向后端注册
3. 设备申请绑定码
4. 伴侣端或管理端确认绑定
5. 设备上传状态与心跳
6. 用户发起互动，后端写入宠物事件与记忆
7. 宠物根据互动变化情绪、经验与成长阶段
8. 设备掉线时记录异常，恢复后触发 `resume` 同步

## 后端职责

- 提供设备注册、绑定与健康查询
- 提供宠物档案、成长、记忆与事件接口
- 维护物种模板、主题包与资源槽位规则
- 根据设备能力返回适合的 `display_mode` 与 `display_hint`

## 设备职责

- 维持心跳
- 上报状态
- 执行基础 UI 渲染
- 按后端返回的模板和能力提示显示宠物

## 验收目标

- 设备注册 → 绑定 → 上报状态链路可跑通
- 宠物创建后可查看档案、成长摘要和记忆摘要
- 设备离线/恢复时，设备日志与宠物日志都有可追踪事件
- 设备能力分级可以为 LCD / OLED / voice / rich 选择合适模板
