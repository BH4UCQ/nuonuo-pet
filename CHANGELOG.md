# Changelog

## 0.1.0

- 初始化 nuonuo-pet 仓库骨架与目录分层
- 定义项目愿景、路线图、交互流程和开发顺序
- 搭建 FastAPI 后端雏形，完成基础协议、设备、宠物、记忆和资源接口
- 搭建 ESP32 固件状态机雏形，覆盖连接、绑定、就绪、睡眠与异常恢复
- 完成设备注册 / 绑定 / 状态链路、设备事件留痕、健康查询与恢复同步
- 完成宠物核心：成长阶段、情绪、事件驱动变化、成长摘要与记忆摘要
- 完成物种模板、模型路由、主题包、资源槽位与预览接口
- 完成设备能力分级与 display_hint 输出，兼容 LCD / OLED / voice / rich 等模式
- 完成绑定流程、恢复流程、宠物-设备关联与端到端自检
- 在工作区内虚拟环境中安装 PlatformIO，并确认 ESP32 固件可以成功编译
- 补充固件最简日志约定、自检脚本与发布准备文档

## 0.2.0（进行中）

- 记忆系统正式化：已接入本地落盘与检索接口
- 模型路由可配置化：默认、备用、回退与原因输出
- 资源包与主题包流程完善：导入、启用、回滚与兼容校验
- 设备端稳定性增强：离线、恢复、低电量、错误恢复路径
- 宠物成长与事件系统增强：偏好形成、物种曲线和事件细化
